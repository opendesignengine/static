Reporting on Sprint 1 Deliverables

Added by J. Simmons over 7 years ago

I am gathering up the materials to present the team's work for the Sprint 1 Review. I believe we will be claiming done on two of three PBIs:

  • YVN-1: Formal EC Review of Yavin.
    • See the Export Control Review
  • YVN-7: Structural analysis of pressure vessel as a Python library. The library should be tested in MTK, producing a document verifying the library. The design should then be implemented in CADQuery.
    • See the PressureVessel class for the implementation of "Structural analysis of pressure vessel as a Python library"
    • See the Pressure Vessel Calculation for Yavin Thruster report for the implementation of "The library should be tested in MTK, producing a document verifying the library." (source code for MTK). I am especially happy to report that the test cases in Section 4 of the verification document show the PressureVessel passes all nine of the test cases selected from my dissertation research.
    • See the Hello World Chamber for the implementation of "The design should then be implemented in CADQuery." (source code for CadQuery)

Additionally, Jeremy and J. decided to run the STL file through a slicer and produced a 3D printed model of the Hello World Chamber (GCODE from MatterControl).

Finally, I do not believe we will be claiming done on YVN-8: Run a Mach 30 volunteer's 3D printer through at least 1 published calibration test and document results in a short report. Juli's work schedule became much busier than anticipated and she did not have time to resolve technical issues with her printer or then complete the calibration test.
